# ADR-0004: Списывать перерасход категории из накоплений

- Status: Accepted
- Date: 2026-04-26

## Context

Накопления по категориям уже хранятся в `SavingsTransfer` и отображаются в
блоке "Накопления" агрегированно по названию категории. При этом пользователь
может накопить деньги на категорию, например "Путешествия", а в одном из
месяцев потратить по одноименной обычной категории больше выделенного бюджета.

Раньше такой перерасход отражался только в расходах месяца и не уменьшал общий
итог накоплений.

## Decision

При пересчете накоплений перерасход обычной категории списывается из
накоплений той же отображаемой категории:

- перерасход считается как `spent - allocated`, если значение больше нуля;
- списание применяется только для обычных категорий, которые не помечены
  `isSaving`, потому что расходы в копящих категориях уже означают пополнение;
- если по этому названию уже есть положительный баланс накоплений, создается
  `SavingsTransfer` за текущий месяц с отрицательной суммой;
- проверка положительного баланса выполняется в валюте текущего бюджета без
  конвертации между валютами;
- существующий контракт `GET /api/budget/savings` сохраняется: отрицательные
  записи участвуют в `totalBank`, `totalCurrencies`, `monthlyTotals`,
  `categoryTotals` и fallback-агрегации из `transfers`.

Схему БД не меняем: `SavingsTransfer.amount` остается целым числом в minor
units, но теперь доменно допускает как пополнения, так и списания.

## Consequences

- Блок "Накопления" уменьшает общий итог на сумму перерасхода категории.
- В помесячной детализации может появиться отрицательная сумма, например
  "Путешествия: -100 EUR", что означает списание из накоплений.
- Ручной пересчет накоплений остается идемпотентным: отрицательная строка за
  месяц пересоздается из текущих расходов и выделенного бюджета.
- Если будущая версия потребует ручные списания без привязки к перерасходу,
  понадобится отдельное решение о модели данных, чтобы не смешивать ручные и
  автоматически рассчитанные движения.
